# 상품 목록 페이지 프로젝트

## 프로젝트 개요

이 프로젝트는 사용자에게 신발 상품 목록을 시각적으로 제공하고, 사용자가 원하는 상품을 장바구니에 담을 수 있는 인터랙티브한 웹 페이지입니다.

* **주요 기능**:
  * 상품 목록 렌더링 (데이터 기반)
  * 장바구니 담기 기능 (중복 담기 방지)
  * 헤더의 장바구니 아이콘을 통한 실시간 수량(Badge) 표시
  * 반응형 그리드 레이아웃 및 애니메이션 효과

- 참여한 팀원: [![GitHub Badge](https://img.shields.io/badge/GitHub-seochaemin13--ui-181717?style=flat-square&logo=GitHub&logoColor=white)](https://github.com/seochaemin13-ui)


## 개발 환경

* **Framework**: React 19.2.0
* **Build Tool**: Vite 7.2.4
* **Language**: JavaScript (ES6+), JSX
* **Styling**: CSS3 (Flexbox, Grid, CSS Container Queries 사용)
* **Deployment**: GitHub Pages (Base path: `/CreateProductListPage/`)


## 설치 및 실행 방법

1. 의존성 라이브러리 설치:
```
npm install
```

2. 개발 서버 실행:
```
npm run dev
```


## 컴포넌트 구조

본 프로젝트는 관심사 분리를 위해 기능별로 컴포넌트와 스타일시트를 나누어 관리합니다.

* **`App.jsx`**: 메인 부모 컴포넌트입니다. 전체 상품 데이터(`cardsData`)를 관리하고, 장바구니의 총 수량(`cartCount`) 상태를 제어하여 헤더와 카드 섹션에 전달합니다.
* **`Card.jsx`**: 개별 상품을 표시하는 재사용 가능한 컴포넌트입니다. 자체적으로 '담김' 상태(`isAdded`)를 관리하여 버튼의 중복 클릭을 방지하고 UI를 변경합니다.
* **`App.css` / `Card.css`**: 레이아웃 및 디자인을 담당합니다. `clamp()` 함수를 사용하여 기기 화면 크기에 따라 폰트 크기가 유동적으로 변하는 반응형 디자인이 적용되어 있습니다.


## 테스트 방법

이 프로젝트는 **Jest**와 **React Testing Library**를 사용하여 구성 요소의 기능과 컴포넌트 간의 상호작용을 검증합니다.

### 테스트 실행 명령어

로컬 환경에서 아래 명령어를 입력하여 전체 테스트 케이스를 실행할 수 있습니다.

```
npm test
```

### 주요 테스트 항목

#### 1. 통합 테스트 (`App.test.js`)

* **상품 목록 렌더링**: 초기 로드 시 `cardsData`에 정의된 상품 개수만큼 카드 컴포넌트가 화면에 올바르게 표시되는지 확인합니다.
* **장바구니 수량 연동**: 특정 상품의 '담기' 버튼을 클릭했을 때, 헤더 영역의 장바구니 배지(Badge) 숫자가 실시간으로 증가하는지 검증합니다.

#### 2. 단위 테스트 (`Card.test.js`)

* **데이터 바인딩**: 전달받은 상품명(title), 가격(price), 설명(description) 등 데이터가 UI 상의 올바른 위치에 렌더링되는지 확인합니다.
* **상태 변경 로직**: 버튼 클릭 시 버튼 텍스트가 **'담김!'**으로 변경되고, 시각적 피드백을 위한 CSS 클래스(`.added`)가 정상적으로 적용되는지 확인합니다.
* **중복 클릭 방지**: 이미 장바구니에 담긴 상품은 추가로 클릭되거나 수량이 중복 합산되지 않도록 이벤트 전파를 차단하는지 검증합니다.
