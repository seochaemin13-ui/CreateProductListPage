## 📝 고객 요구사항 정리
* **페이지 이동:** 상품 목록의 '결제' 버튼 클릭 시, 보유 카드 목록(또는 결제창)으로 이동해야 함.
* **카드 등록(CRUD):**
* 사용자는 새로운 신용카드를 등록할 수 있어야 함.
* 카드는 여러 장 등록이 가능해야 하며, 목록에서 관리되어야 함.
* 카드 별칭(Alias) 설정이 가능해야 함 (선택 사항).


* **UX/UI 인터랙션:**
* 카드 정보 입력 시, **가상의 카드 이미지(Virtual Card)**에 입력된 정보가 실시간으로 반영되어야 함.
* 빈 카드가 있을 때와 등록된 카드가 있을 때의 UI가 구분되어야 함.


* **보안 및 마스킹(Security & Masking):**
* **카드 번호:** 16자리 중 앞 8자리는 표시, 뒤 8자리는 마스킹(`●`) 처리.
* **보안 코드(CVC):** 입력 시 숨김 처리 또는 마스킹.
* **카드 비밀번호:** 앞 2자리 입력 (보안상 마스킹 처리).


* **기타:**
* 테스트 가능한 배포 URL 제공 필요


## 📋 기능 목록
### A. 공통
* **페이지 구조:**
* `/`: 상품 목록 페이지 (Home)
* `/payment`: 보유 카드 목록 페이지
* `/card/add`: 카드 추가 페이지
### B. 상품 목록 페이지 (Product List)
* **'구매하기(결제)' 버튼** 구현.
* 구매 버튼 클릭 시 `/payment` 경로로 이동.
### C. 카드 추가 페이지 (Add Card Form)
* **`<CardDisplay />` (상단 가상 카드 뷰)**
* 사용자가 아래 폼에 입력하는 내용(카드 번호, 만료일, 소유자 이름)을 실시간으로 렌더링.
* 카드사(Bank) 식별 로직 (선택 사항: 번호 앞자리에 따라 색상 변경 등).
* **`<CardForm />` (입력 폼 컨테이너)**
* **`<InputCardNumber />`**:
* 숫자만 입력 가능하도록 제어.
* 4자리마다 하이픈(-) 처리.
* 최대 16자리 제한.
* **`<InputExpirationDate />`**:
* MM / YY 형식 입력.
* 월(1~12), 연도 유효성 검사 (숫자만 입력).
* **`<InputSecurityCode />`**:
* CVC 번호 3자리 입력.
* `type="password"` 속성 사용하여 입력값 숨김.
* **`<InputCardPassword />`**:
* 카드 비밀번호 앞 2자리 입력 필드.
* 각 자릿수 별 별도 input 박스 구현.
### D. 보유 카드 목록 페이지 (My Card List)
* **`<CardList />`**
* **Empty State:** 등록된 카드가 없을 때 "새로운 카드를 등록해주세요" 메시지와 `+` 버튼 표시.
* **List State:** 등록된 카드 리스트를 스크롤 가능한 형태로 표시.
* 각 카드 항목에 '삭제' 기능.
* 특정 카드 선택 시 "이 카드로 결제하기" 액션 활성화.
